Project Architecture
---------------------
Frontend (Angular 17 standalone app):
- Angular standalone components with a login flow, dashboard view, and CPE detail view.
- Uses Router, HttpClient, and Socket.IO for real-time dashboard updates.
- JWT-based auth with token storage in localStorage and route protection by an AuthGuard.
- Core Angular services: CpeService, AuthService, WebSocketService, ThemeService.
- HTTP interceptor attaches Bearer token and handles 401/403 by logout + redirect.

Backend (Node.js + Express + Mongoose + TR-069 ACS):
- Express app exposing three logical route groups:
  - /acs: TR-069 SOAP endpoint for CPEs, protected by Basic Auth.
  - /auth: authentication endpoint for Angular login.
  - /api: authenticated JSON API for dashboard and CPE control, protected by JWT middleware.
- Middleware: helmet, cors, cookie-parser, express-rate-limit, JWT verification, ACS Basic auth.
- Mongoose models for User and Cpe.
- TR-069-specific flow in cwmpController with parsing of Inform, GetParameterValuesResponse, SetParameterValuesResponse.
- Connection Request handling through crService, including IPv6-safe rate-limit key generation.
- Normalization service extracts WAN IP, optical metrics, Wi-Fi metrics, and profile data from CPE parameter lists.
- Structured JSON logging added across server, controllers and middlewares.
